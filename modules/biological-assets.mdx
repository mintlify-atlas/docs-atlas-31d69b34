# Biological Asset Tracking

The Biological Assets module tracks forest plantations and biological assets at **Level 6** - the most granular level representing specific genetic materials planted in Level 4 stands.

## Asset Classification

Biological assets are classified by their primary purpose:

```typescript
enum BiologicalAssetType {
  COMERCIAL      // Commercial production
  INVESTIGACION  // Research/experimental
}
```

## IMA Classification

**IMA (Incremento Medio Anual)** - Mean Annual Increment - classifies growth potential:

```typescript
enum ImaClassification {
  I    // Class I - Highest growth
  II   // Class II
  III  // Class III
  IV   // Class IV
  V    // Class V - Lowest growth
}
```

<Note>
IMA classes help predict yield and plan harvest rotations. Each organization can define custom IMA ranges per class.
</Note>

## Biological Asset Level 6

Level 6 assets represent planted genetic material within a Level 4 stand:

```typescript
type ForestBiologicalAssetLevel6 = {
  id: string;
  level4Id: string;                          // Parent Level 4 stand
  biologicalAssetKey: string;                // Unique key
  accountingKey?: string;                    // Accounting reference
  
  // Establishment Information
  establishmentDate?: Date;
  plantingYear?: number;
  assetType: BiologicalAssetType;
  
  // Genetic Material
  geneticMaterialCode?: string;
  geneticMaterialName?: string;
  
  // Management
  managementSchemeCode?: string;
  managementSchemeName?: string;
  
  // Inventory Data
  inventoryCode?: string;
  inventoryType?: string;
  inventoryDate?: Date;
  inventoryAgeYears?: number;
  level5UnitCount?: number;                   // Sample plots used
  
  // Spacing
  spacingCode?: string;
  spacingDescription?: string;
  spacingBetweenRowsM?: number;               // Meters between rows
  spacingBetweenTreesM?: number;              // Meters between trees
  treeDensityPerHa?: number;                  // Trees per hectare
  
  // Health and Growth
  survivalRate?: number;                      // Percentage (0-100)
  dominantHeightM?: number;                   // Dominant height in meters
  meanHeightM?: number;                       // Mean height in meters
  quadraticDiameterM?: number;                // Quadratic mean diameter
  basalAreaM2?: number;                       // Basal area in m²
  
  // Volume Calculations
  unitVolumeM3NoBarkPerHa?: number;          // Volume without bark per ha
  unitVolumeM3WithBarkPerHa?: number;        // Volume with bark per ha
  totalVolumeM3NoBark?: number;              // Total volume without bark
  totalVolumeM3WithBark?: number;            // Total volume with bark
  adjustedVolumeM3NoBarkPerHa?: number;      // Adjusted volume
  adjustedVolumeM3WithBarkPerHa?: number;
  
  // IMA Classification
  imaClassCode?: string;
  imaClassName?: string;
  
  // Accounting
  actualCostUsd?: number;                    // Actual cost in USD
  
  isActive: boolean;
};
```

## Spacing Configurations

Spacing configurations define tree planting patterns:

```typescript
type Spacing = {
  id: string;
  code: string;
  name: string;
  description?: string;
  betweenRowsM?: number;          // Distance between rows (m)
  betweenTreesM?: number;         // Distance between trees (m)
  treeDensityPerHa?: number;      // Calculated density
  isActive: boolean;
};
```

**Example Spacings:**
- `3x2` → 3 meters between rows, 2 meters between trees
- `4x3` → 1,666 trees/hectare
- `6x6` → 277 trees/hectare

## IMA Classes

IMA (Mean Annual Increment) classes are configured per organization:

```typescript
type ImaClass = {
  id: string;
  code: string;
  classification: ImaClassification;  // I, II, III, IV, V
  name: string;
  description?: string;
  rangeMin?: number;                  // Minimum m³/ha/year
  rangeMax?: number;                  // Maximum m³/ha/year
  isActive: boolean;
};
```

**Example IMA Configuration:**

| Class | Range (m³/ha/year) | Description |
|-------|-------------------|-------------|
| I     | 35+               | Excellent growth |
| II    | 28-35             | Good growth |
| III   | 20-28             | Average growth |
| IV    | 12-20             | Below average |
| V     | < 12              | Poor growth |

## Asset Measurement Campaigns

Biological assets are measured through campaigns (see Audit System for details):

```typescript
type AssetMeasurementCampaign = {
  id: string;

  campaignName: string;
  campaignDate: Date;

  schemaVersion: string;

```

## Code Example

```typescript
// Fetch biological assets for a Level 4 stand
const level4Id = 'uuid-here';

// Note: Level 6 assets are typically created through
// measurement campaigns and calculation jobs

// Fetch species catalog
const speciesResponse = await fetch(
  '/api/forest/config/species?page=1&limit=100'
);
const species = await speciesResponse.json();

// Create vegetal material
const materialResponse = await fetch(
  '/api/forest/config/vegetal-materials',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      code: 'VM001',
      name: 'Pinus caribaea - Clon Elite',
      speciesId: 'species-uuid',
      materialType: 'PURA',
      plantType: 'CLON',
      plantOrigin: 'INTRODUCIDA',
      provenanceId: 'provenance-uuid'
    })
  }
);
```

<Note>
Biological assets at Level 6 are typically generated automatically through measurement campaigns and calculation protocols. Manual creation is rare.
</Note>
