## Overview

SMyEG uses **NextAuth v5** for authentication with a credential-based provider. The system implements secure password verification, account lockout mechanisms, and comprehensive audit logging.

## Authentication Configuration

The authentication system is configured in `src/lib/auth.ts` using NextAuth v5 with JWT session strategy:

```typescript src/lib/auth.ts
import NextAuth from "next-auth";
import Credentials from "next-auth/providers/credentials";

export const { handlers, auth, signIn, signOut } = NextAuth({
  trustHost: true,
  session: { strategy: "jwt" },
  pages: {
    signIn: "/login",
  },
  providers: [Credentials({ /* ... */ })],
});
```

### Key Configuration Options

- **Session Strategy**: JWT tokens for stateless authentication
- **Trust Host**: Enabled for multi-organization deployment
- **Custom Sign-In Page**: `/login`

## Credential Provider

The credential provider accepts three fields:

<ParamField path="identifier" type="string" required>
  Username or email address (case-insensitive)
</ParamField>

<ParamField path="password" type="string" required>
  User password (verified with bcrypt)
</ParamField>

<ParamField path="organizationId" type="string" required>
  Organization ID for multi-tenant context
</ParamField>

## Authorization Flow

The authorization process follows these steps:

<Steps>
  <Step title="Credential Validation">
    Validates that identifier, password, and organizationId are provided.
  </Step>
  
  <Step title="User Lookup">
    Searches for user by email OR username (case-insensitive):
    
    ```typescript
    const user = await prisma.user.findFirst({
      where: {
        OR: [
          { email: { equals: identifier, mode: "insensitive" } },
          { username: { equals: identifier, mode: "insensitive" } }
        ]
      },
      include: { organization: true }
    });
    ```
  </Step>
  
  <Step title="Account Status Check">
    Verifies:
    - User has a password hash
    - Account status is `ACTIVE`
    - Account is not locked (`lockedUntil` is in the past or null)
  </Step>
  
  <Step title="Password Verification">
    Uses bcrypt to verify the password against the stored hash.
  </Step>
  
  <Step title="Failed Login Handling">
    On invalid password:
    - Increments `failedLoginAttempts`
    - Locks account for 30 minutes after 5 failed attempts
    - Creates `LOGIN_FAILED` audit log entry
    
    ```typescript src/lib/auth.ts
    const attempts = user.failedLoginAttempts + 1;
    await prisma.user.update({
      where: { id: user.id },
      data: {
        failedLoginAttempts: attempts,
        lockedUntil: attempts >= 5 
          ? new Date(Date.now() + 30 * 60 * 1000) 
          : null
      }
    });
    ```
  </Step>
  
  <Step title="Organization Authorization">
    Fetches authorized organizations for the user and validates the selected organizationId.
  </Step>
  
  <Step title="Successful Login">
    On success:
    - Resets `failedLoginAttempts` to 0
    - Updates `lastLoginAt` timestamp
    - Creates `LOGIN` audit log entry
    - Returns user object with roles and permissions
  </Step>
</Steps>

## Session Management

### JWT Callback

Enriches the JWT token with user metadata:

```typescript src/lib/auth.ts
async jwt({ token, user }) {
  if (user) {
    token.id = user.id;
    token.roles = user.roles;
    token.permissions = user.permissions;
    token.organizationId = user.organizationId ?? null;
    token.organizationName = user.organizationName ?? null;
  }
  return token;
}
```

### Session Callback

Maps JWT token data to the session object:

```typescript src/lib/auth.ts
async session({ session, token }) {
  if (session.user) {
    session.user.id = token.id;
    session.user.roles = token.roles ?? [];
    session.user.permissions = token.permissions ?? [];
    session.user.organizationId = token.organizationId ?? null;
    session.user.organizationName = token.organizationName ?? null;
  }
  return session;
}
```

## Accessing the Current Session

### Server Components

```typescript
import { auth } from "@/lib/auth";

export default async function ServerPage() {
  const session = await auth();
  
  if (!session?.user?.id) {
    redirect("/login");
  }
  
  return <div>Welcome {session.user.name}</div>;
}
```

### API Routes

```typescript
import { auth } from "@/lib/auth";

export async function GET() {
  const session = await auth();
  
  if (!session?.user?.id) {
    return new Response("Unauthorized", { status: 401 });
  }
  
  // Your API logic here
}
```

### Client Components

Use `useSession` from next-auth/react:

```typescript
"use client";

import { useSession } from "next-auth/react";

export function ClientComponent() {
  const { data: session, status } = useSession();
  
  if (status === "loading") return <div>Loading...</div>;
  if (!session) return <div>Not authenticated</div>;
  
  return <div>Welcome {session.user.name}</div>;
}
```

## Sign In and Sign Out

### Programmatic Sign In

```typescript
import { signIn } from "@/lib/auth";

await signIn("credentials", {
  identifier: "user@example.com",
  password: "securePassword",
  organizationId: "org-id-123",
  redirectTo: "/dashboard"
});
```

### Sign Out

```typescript
import { signOut } from "@/lib/auth";

await signOut({ redirectTo: "/login" });
```

<Note>
  The authentication system automatically handles organization context, ensuring users only access data within their authorized organizations.
</Note>
