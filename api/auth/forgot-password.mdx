## POST /api/auth/forgot-password

Initiates the password reset process by generating a secure reset token. If the email exists, a token is created that can be used to reset the password.

### Request Body

<ParamField body="email" type="string" required>
  Valid email address registered in the system
</ParamField>

### Response

#### Success Response (200)

```json
{
  "success": true,
  "data": {
    "message": "Si el correo existe, se enviará un enlace de recuperación."
  }
}
```

<ResponseField name="message" type="string">
  Generic success message to prevent user enumeration
</ResponseField>

#### Error Response (400)

```json
{
  "success": false,
  "message": "Datos inválidos",
  "error": {
    "fieldErrors": {
      "email": ["Invalid email"]
    }
  }
}
```

### Security Features

**User Enumeration Protection**: The endpoint always returns the same success message regardless of whether the email exists. This prevents attackers from discovering valid email addresses.

**Token Security**:
- Tokens are cryptographically secure random strings
- Stored as SHA-256 hashes in the database
- Valid for 60 minutes (1 hour)
- Single-use only (marked as used after password reset)

**Email Notification** (Implementation Note): While the code generates the token, the actual email sending is not shown in this route. Implement email delivery via your preferred email service.

### Token Generation

When a valid user is found:

1. **Generate Token**: Creates a secure random token
2. **Hash Token**: Token is hashed using SHA-256 before storage
3. **Store in Database**: Creates `PasswordResetToken` record with:
   - `userId`: Owner of the token
   - `tokenHash`: Hashed version of the token
   - `expiresAt`: Current time + 60 minutes
4. **Send Email**: Email the unhashed token to the user (implement in your email service)

### Example

<CodeGroup>

```bash cURL
curl -X POST 'https://api.smyeg.com/api/auth/forgot-password' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "user@example.com"
  }'
```

```javascript JavaScript
async function requestPasswordReset(email) {
  const response = await fetch('https://api.smyeg.com/api/auth/forgot-password', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ email })
  });

  const result = await response.json();
  
  if (result.success) {
    console.log(result.data.message);
    // Show user a message to check their email
  } else {
    console.error('Invalid email format');
  }
}

await requestPasswordReset('user@example.com');
```

```python Python
import requests

def request_password_reset(email: str):
    response = requests.post(
        'https://api.smyeg.com/api/auth/forgot-password',
        json={'email': email}
    )
    
    data = response.json()
    if data['success']:
        print(data['data']['message'])
        # Instruct user to check their email
    else:
        print('Invalid email format')

request_password_reset('user@example.com')
```

</CodeGroup>

### Password Reset Flow

1. **Request Reset**: User calls `/api/auth/forgot-password` with email
2. **Check Email**: User receives email with reset token (if email exists)
3. **Reset Password**: User calls `/api/auth/reset-password` with token and new password
4. **Sign In**: User can now sign in with new password

### Token Properties

<ParamField body="token" type="string">
  Secure random string sent to user's email
</ParamField>

<ParamField body="tokenHash" type="string">
  SHA-256 hash stored in database
</ParamField>

<ParamField body="expiresAt" type="datetime">
  Token expiration time (60 minutes from creation)
</ParamField>

<ParamField body="usedAt" type="datetime">
  Timestamp when token was used (null until used)
</ParamField>

### Implementation Notes

**Email Integration**: You need to integrate an email service to send the reset token to users. The token should be sent as part of a reset link:

```
https://yourdomain.com/reset-password?token=GENERATED_TOKEN
```

**Token Cleanup**: Consider implementing a cleanup job to remove expired tokens from the database.

**Rate Limiting**: Implement rate limiting on this endpoint to prevent abuse:
- Limit requests per IP address
- Limit requests per email address
- Add CAPTCHA for suspicious activity
