## POST /api/auth/register

Creates a new user account with automatic default organization assignment and role provisioning.

### Request Body

<ParamField body="email" type="string" required>
  Valid email address (must be unique)
</ParamField>

<ParamField body="password" type="string" required>
  Password meeting security requirements:
  - Minimum 8 characters
  - At least one uppercase letter
  - At least one lowercase letter
  - At least one number
</ParamField>

<ParamField body="confirmPassword" type="string" required>
  Must match the password field
</ParamField>

<ParamField body="firstName" type="string" required>
  User's first name (minimum 2 characters)
</ParamField>

<ParamField body="lastName" type="string" required>
  User's last name (minimum 2 characters)
</ParamField>

### Response

#### Success Response (201)

```json
{
  "success": true,
  "data": {
    "message": "Cuenta creada. Pendiente de aprobacion por un ADMIN."
  }
}
```

<ResponseField name="message" type="string">
  Confirmation message indicating account creation is pending admin approval
</ResponseField>

#### Error Responses

**400 Bad Request** - Validation errors

```json
{
  "success": false,
  "message": "Datos inválidos",
  "error": {
    "fieldErrors": {
      "email": ["Invalid email"],
      "password": ["Debe contener al menos una mayúscula"]
    }
  }
}
```

**409 Conflict** - Email already exists

```json
{
  "success": false,
  "message": "El email ya existe"
}
```

### Registration Process

1. **Validation**: Request body is validated against the registration schema
2. **Email Check**: System verifies email is not already registered
3. **Organization Assignment**: User is assigned to default organization ("Por defecto")
4. **Role Assignment**: User receives default "USER" role with permissions
5. **Password Hashing**: Password is securely hashed using bcrypt
6. **User Creation**: User record is created with status `PENDING_VERIFICATION`
7. **Audit Log**: Registration action is logged for security tracking

### Default Configuration

- **Initial Status**: `PENDING_VERIFICATION` - requires admin approval
- **Default Organization**: "Por defecto" organization (created if doesn't exist)
- **Default Role**: USER role with basic permissions
- **Organization Slug**: Generated from organization name

### Password Requirements

Passwords must meet the following criteria:

- Minimum length: 8 characters
- At least one uppercase letter (A-Z)
- At least one lowercase letter (a-z)
- At least one digit (0-9)

### Post-Registration

After successful registration:

1. **Admin Approval Required**: User status remains `PENDING_VERIFICATION` until an admin activates the account
2. **Cannot Login**: Users cannot authenticate until their status is changed to `ACTIVE`
3. **Audit Trail**: Registration is logged in the audit log system
