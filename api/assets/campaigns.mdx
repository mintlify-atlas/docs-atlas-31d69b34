# Measurement Campaigns API

Manage measurement campaigns for tracking biological, hydrological, dendrometric, and demographic assets across forest patrimony units.

## Overview

Measurement campaigns organize field observations under specific protocols and dates. Each campaign is associated with a Level 4 forest patrimony unit and follows a structured measurement protocol.

### Domain Types

- `BIOLOGICO` - Biological assets (survival, growth, health)
- `HIDROLOGICO` - Hydrological monitoring (flow, precipitation)
- `DENDROMETRICO` - Dendrometric measurements (volume, diameter)
- `DEMOGRAFICO` - Demographic surveys (community, household)

### Protocol Types

**Biological Domain:**
- `BIO_SOBREVIVENCIA` - Survival rate assessment
- `BIO_PREOPERATIVO` - Pre-operational inventory
- `BIO_PRECOSECHA` - Pre-harvest evaluation
- `BIO_CONTINUO` - Continuous monitoring
- `BIO_PARCELAS_PERMANENTES` - Permanent plots

**Hydrological Domain:**
- `HIDRO_ESTACION_METEOROLOGICA` - Weather station
- `HIDRO_ESTACION_HIDROLOGICA` - Hydrological station

**Dendrometric Domain:**
- `DENDRO_PARCELA_MULTIPROPOSITO` - Multi-purpose plots
- `DENDRO_PARCELA_CARBONO` - Carbon sequestration plots

**Demographic Domain:**
- `DEMO_ENCUESTA_COMUNIDAD` - Community survey
- `DEMO_ENCUESTA_NUCLEO_FAMILIAR` - Household survey

## Create Campaign

<api-endpoint method="POST" path="/api/assets/measurement-campaigns">
Create a new measurement campaign.
</api-endpoint>

### Request Body

<ParamField body="level4Id" type="uuid" required>
  Forest patrimony Level 4 unit ID
</ParamField>

<ParamField body="domain" type="enum" required>
  Asset domain: `BIOLOGICO`, `HIDROLOGICO`, `DENDROMETRICO`, `DEMOGRAFICO`
</ParamField>

<ParamField body="protocolType" type="enum" required>
  Measurement protocol type (see Protocol Types section)
</ParamField>

<ParamField body="campaignCode" type="string" required>
  Unique campaign code (1-120 chars)
</ParamField>

<ParamField body="campaignName" type="string" required>
  Campaign display name (2-255 chars)
</ParamField>

<ParamField body="campaignDate" type="datetime" required>
  Campaign reference date
</ParamField>

<ParamField body="schemaVersion" type="string" default="1.0">
  Data schema version (max 32 chars)
</ParamField>

<ParamField body="status" type="enum" default="DRAFT">
  Initial campaign status: `DRAFT`, `ACTIVE`, `CLOSED`, `CANCELLED`
</ParamField>

<ParamField body="notes" type="string">
  Additional notes or comments
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata JSON object
</ParamField>

### Response

Returns the created campaign object with HTTP status `201 Created`.

<Note>
  The campaign code must be unique within the organization. The Level 4 unit must belong to the user's organization (unless super admin).
</Note>

### Example Request

```bash
curl -X POST "https://api.example.com/api/assets/measurement-campaigns" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level4Id": "level4-uuid",
    "domain": "BIOLOGICO",
    "protocolType": "BIO_SOBREVIVENCIA",
    "campaignCode": "SURV-2026-Q1",
    "campaignName": "Survival Assessment Q1 2026",
    "campaignDate": "2026-03-15",
    "status": "ACTIVE",
    "notes": "6-month post-planting assessment"
  }'
```

## Update Campaign

<api-endpoint method="PATCH" path="/api/assets/measurement-campaigns">
Update an existing measurement campaign.
</api-endpoint>

### Request Body

<ParamField body="id" type="uuid" required>
  Campaign ID to update
</ParamField>

<ParamField body="campaignName" type="string">
  Updated campaign name (2-255 chars)
</ParamField>

<ParamField body="campaignDate" type="datetime">
  Updated campaign date
</ParamField>

<ParamField body="schemaVersion" type="string">
  Updated schema version
</ParamField>

<ParamField body="status" type="enum">
  Updated status: `DRAFT`, `ACTIVE`, `CLOSED`, `CANCELLED`
</ParamField>

<ParamField body="notes" type="string">
  Updated notes
</ParamField>

<ParamField body="metadata" type="object">
  Updated metadata
</ParamField>

<Note>
  At least one field besides `id` must be provided. Campaign code, domain, protocol type, and level4Id cannot be changed after creation.
</Note>

### Response

Returns the updated campaign object.

### Example Request

```bash
curl -X PATCH "https://api.example.com/api/assets/measurement-campaigns" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "campaign-uuid",
    "status": "CLOSED",
    "notes": "Campaign completed successfully"
  }'
```

## Delete Campaign

<api-endpoint method="DELETE" path="/api/assets/measurement-campaigns">
Delete a measurement campaign.
</api-endpoint>

### Request Body

<ParamField body="id" type="uuid" required>
  Campaign ID to delete
</ParamField>

```json

}
```

## Permissions

All campaign endpoints require authentication and appropriate permissions:

- **GET**: Requires `asset-measurement:READ` or any write permission
- **POST**: Requires `asset-measurement:CREATE`
- **PATCH**: Requires `asset-measurement:UPDATE`
- **DELETE**: Requires `asset-measurement:DELETE`

Super admins bypass organization restrictions.

## Audit Logging

All create, update, and delete operations are logged to the audit log with:
- User ID
- Action type (CREATE, UPDATE, DELETE)
- Entity type: `AssetMeasurementCampaign`
- Entity ID
- Changed values
