# Measurement Observations API

Record and manage field measurement observations within campaigns. Each observation contains protocol-specific data collected from forest patrimony units.

## Overview

Observations are individual measurement records associated with a campaign. The payload structure varies by protocol type and contains the actual field measurements.

### Observation Structure

- **campaignId**: Links to measurement campaign
- **level5UnitCode**: Optional reference to Level 5 sampling unit
- **sectionKey**: Optional section identifier for spatial organization
- **observedAt**: Measurement timestamp
- **payload**: JSON object with protocol-specific measurements

## Create Observation

<api-endpoint method="POST" path="/api/assets/measurement-observations">
Create a new measurement observation.
</api-endpoint>

### Request Body

<ParamField body="campaignId" type="uuid" required>
  Campaign ID to associate observation with
</ParamField>

<ParamField body="level5UnitCode" type="string">
  Level 5 sampling unit code
</ParamField>

<ParamField body="sectionKey" type="string">
  Section identifier for spatial organization
</ParamField>

<ParamField body="observedAt" type="datetime">
  Measurement timestamp
</ParamField>

<ParamField body="payload" type="object" required>
  Protocol-specific measurement data (JSON object)
</ParamField>

### Protocol Validation

For `BIO_SOBREVIVENCIA` protocol, the payload is validated against specific business rules:

**Required Fields:**
- `lineId` (integer > 0)
- `sampleUnitId` (integer > 0)
- `spacingL1` (number > 0)
- `spacingL2` (number > 0)
- `liveNormal` (number >= 0)
- `missingCount` (number >= 0)
- `avgHeight` (number >= 0)

**Optional Fields:**
- `liveAtipic` (number >= 0, default 0)
- `liveDrought` (number >= 0, default 0)
- `livePest` (number >= 0, default 0)
- `deadPest` (number >= 0, default 0)
- `deadDrought` (number >= 0, default 0)
- `deadMech` (number >= 0, default 0)

**Business Rule:**
The sum of live + dead + missing must match the theoretical density:
```
evaluatedPoints = liveTotal + deadTotal + missingCount
expectedPoints = 100 / (spacingL1 * spacingL2)
|evaluatedPoints - expectedPoints| <= 1
```

<Note>
  Other protocols have their own validation rules. Generic protocols accept any valid JSON payload.
</Note>

### Response

Returns the created observation object with HTTP status `201 Created`.

## Permissions

All observation endpoints require authentication and appropriate permissions:

- **GET**: Requires `asset-measurement:READ` or any write permission
- **POST**: Requires `asset-measurement:CREATE`
- **PATCH**: Requires `asset-measurement:UPDATE`
- **DELETE**: Requires `asset-measurement:DELETE`

Super admins bypass organization restrictions.

## Audit Logging

All create, update, and delete operations are logged to the audit log with:
- User ID
- Action type (CREATE, UPDATE, DELETE)
- Entity type: `AssetMeasurementObservation`
- Entity ID
- Changed values
