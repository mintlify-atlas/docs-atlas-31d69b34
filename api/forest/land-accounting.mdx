## Overview

The Land Accounting API tracks patrimonial variations resulting from land use changes at Level 4 units. Each variation records:

- **Land Use Transition**: Previous → New land use classification
- **Area**: Affected area in hectares
- **Valuation**: Reference values before/after, total values, patrimonial delta (USD)
- **Classification**: Kind (INCREMENTO, DECREMENTO, SIN_CAMBIO)
- **Workflow**: Status (PENDIENTE, PROCESADA, ANULADA)

Variations are typically generated automatically during geo import operations or can be triggered by land use changes.

## Create Land Variation

<CodeGroup>
```bash POST /api/forest/land-accounting/variations
curl -X POST "https://api.smyeg.com/api/forest/land-accounting/variations" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{}'
```
</CodeGroup>

<Warning>
  Direct creation is **not supported** (returns 405 Method Not Allowed).
  
  Land variations are automatically generated from:
  - Geo import operations (shapefile processing)
  - Level 4 land use updates via patrimony API
  - Geo operation jobs (split/merge)
</Warning>

## Variation Logic

Variations are created when:

1. **Geo Import**: Shapefile `usoactual` ≠ Level 4 `currentLandUseName`
2. **Manual Update**: PATCH `/api/forest/patrimony` with changed `currentLandUseName`
3. **Geo Operations**: Split/merge operations that modify land use

### Calculation Rules

```typescript
// Area from geometry
affectedAreaHa = ST_Area(geometry_utm) / 10000

// Values (simplified - actual logic uses valuation tables)
totalValueBeforeUsd = affectedAreaHa * referenceValueBeforeUsd
totalValueAfterUsd = affectedAreaHa * referenceValueAfterUsd
patrimonialDeltaUsd = totalValueAfterUsd - totalValueBeforeUsd

// Kind classification
if (patrimonialDeltaUsd > 0) kind = "INCREMENTO"
else if (patrimonialDeltaUsd < 0) kind = "DECREMENTO"
else kind = "SIN_CAMBIO"
```

### Status Workflow

<Steps>
  <Step title="PENDIENTE">
    Created automatically, awaiting review
  </Step>
  
  <Step title="PROCESADA">
    Approved and applied to accounting (manual process)
  </Step>
  
  <Step title="ANULADA">
    Cancelled or rejected (manual process)
  </Step>
</Steps>

## Date Range Filtering

The `fromDate` and `toDate` parameters filter by `variationDate` with inclusive bounds:

- `fromDate`: Sets time to 00:00:00.000
- `toDate`: Sets time to 23:59:59.999

<CodeGroup>
```bash Example: Q1 2024 variations
curl -X GET "https://api.smyeg.com/api/forest/land-accounting/variations?fromDate=2024-01-01&toDate=2024-03-31" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

## Reporting

Variations support financial and environmental reporting:

### Financial Impact
- **Incremental Value**: Sum of `patrimonialDeltaUsd` where `kind = INCREMENTO`
- **Decremental Value**: Sum of `patrimonialDeltaUsd` where `kind = DECREMENTO`
- **Net Change**: Total `patrimonialDeltaUsd` across all variations

### Land Use Transition Matrix
Track transitions between land use types:
```
PASTIZAL → BOSQUE PLANTADO: 120.5 ha (+$421,750)
BOSQUE NATURAL → CONSERVACIÓN: 85.3 ha ($0)
```

## Integration Points

### Geo Import
When importing shapefiles via `/api/forest/geo/import`:
1. Worker processes each feature
2. Compares `usoactual` (shapefile) vs. `currentLandUseName` (database)
3. If different, creates variation using `variationDate` and `variationNotes` from job metadata
4. Updates Level 4 `currentLandUseName` to match shapefile

### Patrimony Updates
When updating Level 4 via `/api/forest/patrimony`:
1. Detects `currentLandUseName` change
2. Creates variation with current date
3. Uses geometry area from active `forest_geometry_n4` record

## Permissions

Requires:
- `forest-patrimony:READ` (all users can view variations)
- `SUPER_ADMIN` role (cross-organization access)

<Note>
  Processing variations (status changes) requires additional accounting permissions (not covered by this API endpoint).
</Note>
