## First Login

After completing the [installation](/installation), you can log in with the default administrator credentials created during database seeding.

<Steps>
  <Step title="Navigate to the application">
    Open your browser and navigate to `http://localhost:3000`
  </Step>

  <Step title="Select your organization">
    Choose "Default Organization" from the organization selection screen
  </Step>

  <Step title="Enter credentials">
    Use the default admin credentials:
    - **Email**: `admin@example.com`
    - **Password**: `Admin1234`
  </Step>

  <Step title="Access the dashboard">
    After successful authentication, you'll be redirected to the main dashboard
  </Step>
</Steps>

<Warning>
  Change the default admin password immediately after first login. Navigate to your profile settings to update credentials.
</Warning>

## Common First Tasks

### Create Your First Organization

<Steps>
  <Step title="Navigate to Organizations">
    Click **Organizations** in the sidebar navigation
  </Step>

  <Step title="Create new organization">
    Click the **Create Organization** button
  </Step>

  <Step title="Fill in details">
    Provide the organization details:
    - Organization name
    - Tax ID (RIF)
    - Country
    - Contact information
  </Step>

  <Step title="Save and configure">
    Save the organization and configure additional settings like logo and branding
  </Step>
</Steps>

### Add Users to Your Organization

<Steps>
  <Step title="Go to User Management">
    Navigate to **Users** from the sidebar
  </Step>

  <Step title="Create a new user">
    Click **Create User** and provide:
    - Email address
    - Username
    - Full name
    - Initial password
  </Step>

  <Step title="Assign a role">
    Select an appropriate role:
    - **ADMIN**: Full administrative access
    - **GERENTE_CAMPO**: Field manager access
    - **CONTADOR**: Accounting and reporting access
    - **USER**: Basic access
  </Step>

  <Step title="Set organization">
    Assign the user to your organization
  </Step>
</Steps>

### Import Geospatial Data

<Steps>
  <Step title="Prepare your shapefile">
    Create a ZIP file containing:
    - `.shp` (geometry)
    - `.shx` (shape index)
    - `.dbf` (attributes)
    - `.prj` (projection)

    Required attributes:
    - `nivel2_id`: Level 2 ID (finca/property)
    - `nivel3_id`: Level 3 ID (lote/compartment)
    - `nivel4_id`: Level 4 ID (rodal/plot)
  </Step>

  <Step title="Navigate to Forest Patrimony">
    Go to **Forest Patrimony** in the main navigation
  </Step>

  <Step title="Upload shapefile">
    Click **Import Shapefile** and select your ZIP file
  </Step>

  <Step title="Monitor import progress">
    The import runs as a background job. Check the status in the import jobs list.
  </Step>
</Steps>

<Note>
  For detailed information about shapefile requirements and validation rules, see the [Shapefile Import Guide](/geo/shapefile-import).
</Note>

### View Audit Logs

Monitor all system activity through the comprehensive audit system:

<Steps>
  <Step title="Access audit logs">
    Navigate to **Audit** from the sidebar
  </Step>

  <Step title="Filter logs">
    Use filters to find specific events:
    - Date range
    - User
    - Action type (CREATE, UPDATE, DELETE, LOGIN, etc.)
    - Module
  </Step>

  <Step title="View details">
    Click on any audit entry to see:
    - Old and new values
    - Request metadata (IP, user agent)
    - Execution time
    - HTTP status
  </Step>

  <Step title="Export data">
    Export audit logs to CSV or JSON for compliance reporting
  </Step>
</Steps>

## Start Background Workers

For production use, start the background workers to process geospatial jobs and asset measurements:

<CodeGroup>

```bash Development
# Start the geo worker
npm run worker:geo

# In a separate terminal, start the asset measurement worker
npm run worker:assets
```

```bash Production (PM2)
# Start all services including workers
pm2 start ecosystem.config.cjs

# View status
pm2 status

# View logs
pm2 logs confor-geo-worker
```

</CodeGroup>

The workers handle:
- Shapefile imports and validation
- Surface area recalculations
- Land use variation detection
- Asset measurement calculations
